# 1. Project Context

We are building the marketing website for an AI consulting and AI software services company. The website must clearly communicate what we do, who we serve, and why clients should choose us. It should position us as a credible, modern, results-driven AI partner — not a generic IT shop.

## 1.1 Business Positioning

- **Company type:** AI Consulting + Custom AI Software Development
- **Target clients:** Mid-market and enterprise companies (50–500+ employees) looking to adopt, scale, or optimize AI
- **Key differentiator:** We combine strategic consulting with hands-on engineering — from AI roadmap to production deployment
- **Tone of voice:** Confident, clear, jargon-light, results-oriented. Think "smart colleague" not "corporate brochure."
- **Avoid:** Buzzword overload, vague promises, stock-photo energy, "leverage synergies" language

## 1.2 Brand Voice Guidelines

Every piece of copy on this site should feel like it was written by someone who actually builds AI systems, not by a marketing department that Googled "AI trends." Here are the rules:

- Lead with outcomes, not technology. Say what the client gets, then explain how.
- Be specific. Replace "cutting-edge AI solutions" with "custom ML models that reduced churn by 23%."
- Use short sentences. Break complex ideas into digestible pieces.
- Write at a 9th-grade reading level. If a VP of Operations can't scan it in 10 seconds, rewrite it.
- Every page must answer: What is this? Who is it for? What do I do next?

## 1.3 Sitemap & Page Architecture

The website consists of the following pages. Each page's full copy is provided in Part B.

- Homepage (hero, services overview, social proof, CTA)
- Services (main services hub page)
- Service detail pages (6 sub-pages, one per service category)

- Case Studies (project showcases)

# 2. Technical Architecture

## 2.1 Recommended Tech Stack

- **Framework:** Next.js 14+ (App Router) with TypeScript
- **Styling:** Tailwind CSS with custom design tokens
- **Animations:** Framer Motion for page transitions and scroll-triggered reveals
- **CMS:** MDX for blog posts (or Sanity/Contentful if headless CMS is preferred)
- **Forms:** React Hook Form + Zod validation, submitted via API route to email service
- **Scheduling:** Embedded Calendly or Cal.com widget
- **Deployment:** Vercel (recommended) or AWS Amplify
- **Analytics:** Vercel Analytics + Google Analytics 4 + Hotjar for heatmaps

## 2.2 Project Structure

When generating the codebase, use this file structure:

```
/app
  /page.tsx                → Homepage
  /services/page.tsx       → Services hub
  /services/[slug]/page.tsx → Service detail pages
  /industries/page.tsx     → Industries
  /case-studies/page.tsx   → Case studies listing
  /about/page.tsx          → About us
  /blog/page.tsx           → Blog listing
  /contact/page.tsx        → Contact & booking
  /approach/page.tsx       → Our Approach (NEW)
/components               → Shared UI components
/lib                      → Utilities, constants, data
/content                  → MDX blog posts & service data
```

## 2.3 Design System Tokens

Define these as CSS custom properties and Tailwind extend values:

- **Primary (Navy):** #0F2B46 — headings, navigation, footer background
- **Accent (Blue):** #2980B9 — CTAs, links, interactive elements, highlights
- **Accent Light:** #3498DB — hover states, secondary buttons
- **Success Green:** #27AE60 — metrics, positive indicators
- **Background Light:** #F8F9FA — section alternation
- **Background Blue:** #EBF5FB — feature highlights, callout boxes
- **Text Primary:** #2C3E50 — body copy
- **Text Secondary:** #5D6D7E — captions, metadata
- **Border:** #D5D8DC — cards, dividers

**Font — Headings:** Use a distinctive, modern sans-serif such as Satoshi, General Sans, or Cabinet Grotesk (available from Fontshare or Google Fonts). Avoid Inter, Roboto, or Arial.

**Font — Body:** Pair with a clean, readable sans-serif like DM Sans or Plus Jakarta Sans at 16–18px.

**Font — Monospace:** JetBrains Mono or Fira Code for any code snippets or tech labels.

# 3. Design Direction

## 3.1 Overall Aesthetic

The design should feel like a premium, modern tech company — not a generic corporate site and not a Silicon Valley startup template. Think: clean spatial composition with intentional asymmetry, generous white space, and confident typography.

- Dark navigation with light content sections, alternating slightly tinted backgrounds to create visual rhythm
- Card-based layouts for services, case studies, and team members with subtle hover animations
- Geometric accent elements: thin lines, dots, or abstract node/network patterns (not generic AI brain/robot imagery)
- Data visualization motifs: subtle graphs, connection lines, or grid patterns as decorative elements
- Full-bleed CTA sections with contrasting dark background to break up the page and create urgency

## 3.2 Component Specifications

### Navigation *(UPDATED)*

- Sticky header with logo, nav links (Services, **Our Approach**, Industries, Case Studies, About, Blog), and a highlighted "Book a Call" button
- Mega menu for Services dropdown showing all 6 categories with icons — **note:** "AI Strategy & Consulting" is now renamed to "AI Strategy & Business Consulting"
- **Our Approach (new nav item):** Links to a new standalone page at `/approach` or anchors to the methodology section on the About page
- Mobile: hamburger menu with full-screen overlay

### Hero Sections

- Each page has a hero: large headline (48–64px), subheadline (18–22px), and 1–2 CTAs
- Homepage hero: optional subtle animated background (gradient mesh, particle field, or abstract data flow)
- Interior page heroes: cleaner, more compact, with breadcrumb navigation

### Service Cards

- Icon + Title + Description + Arrow link
- Consistent card height using flexbox
- Hover: subtle elevation change, icon color shifts to accent
- Desktop: 3 per row | Tablet: 2 | Mobile: 1

### Stat / Metric Counters

- Large bold number with animated count-up on scroll into view
- Small descriptor text below each number
- 4 stats in a row on desktop, 2×2 on mobile

### Testimonial Cards

- Quote text, client name, title, company, optional photo
- Subtle quotation mark decorative element
- Carousel or grid layout with auto-scroll

### Footer

- 4-column layout: Company info + Services links + Resources links + Contact details
- Newsletter signup inline in footer
- Social media icons, privacy policy, terms links
- Dark background matching navigation

## 3.3 Responsive Breakpoints

- **Desktop:** ≥ 1280px — full layout, 3-column grids, mega menus
- **Tablet:** 768px–1279px — 2-column grids, condensed nav
- **Mobile:** < 768px — single column, hamburger nav, stacked CTAs

## 3.4 Animation Guidelines

Animations should enhance, not distract. They should feel professional and purposeful.

- Page load: staggered fade-up for hero elements (headline → subheadline → CTAs at 100ms intervals)
- Scroll reveal: sections fade in and slide up 20px when entering viewport, triggered once
- Stat counters: animated count from 0 to target number over 1.5 seconds on viewport entry
- Hover states: 200ms ease transitions for color, shadow, and transform changes
- Page transitions: subtle fade between routes (150–200ms)
- No parallax scrolling, no auto-playing videos, no animations that trigger on every scroll

## 3.5 Performance Requirements

- Lighthouse score: 90+ across all categories
- First Contentful Paint: < 1.5 seconds
- Cumulative Layout Shift: < 0.1
- All images: WebP format with lazy loading and srcset for responsive sizes
- Fonts: preloaded, with font-display: swap and appropriate subset

## 3.6 SEO Requirements

- Each page: unique meta title (< 60 chars), meta description (< 155 chars), and Open Graph tags
- Semantic HTML: proper heading hierarchy (one H1 per page), landmark elements, alt text on all images
- Structured data: Organization schema on homepage, Service schema on service pages, Article schema on blog posts
- XML sitemap generated automatically by Next.js
- Canonical URLs on all pages
